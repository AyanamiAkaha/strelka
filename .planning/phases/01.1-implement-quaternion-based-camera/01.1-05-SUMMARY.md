## Files Created/Modified

- `src/core/Math.ts` - Export mat4 from gl-matrix

## Decisions Made

- **Full view matrix approach**: Compute MVP matrix in CPU using mat4.lookAt() with quaternion-derived up vector
- **Matrix multiplication order**: MVP = Projection × View × Model (right-to-left in gl-matrix)
- **Rotation direction fix**: Negate both pitchChange and yawChange to invert rotation direction
- **Speed calibration**: Reduce mouseSensitivity to match pre-quaternion rotation speed

## Deviations from Plan

### Auto-fixed Issues

**1. WebGLPlayground.vue incompatible with Camera.ts API**
- **Found during:** Initial testing (TypeError: camera.value.rotation is undefined)
- **Issue:** WebGLPlayground.vue accessing `camera.value.rotation.x/y` which no longer exists
- **Fix:** Updated to use `camera.value.getShaderUniforms()` which returns quaternion-compatible data
- **Committed in:** `ff57d44`

**2. Shader using fixed world up caused gimbal lock**
- **Found during:** User testing (movement not in camera local space)
- **Issue:** Shader computed view matrix from Euler angles with fixed world up `[0, 1, 0]`
- **Root cause:** ShaderManager.ts line 222: `vec3 up = vec3(0.0, 1.0, 0.0);`
- **Fix:** Switched to CPU-computed view matrix using `mat4.lookAt()` with quaternion-derived up vector
- **Committed in:** `26c7f74`

**3. Inverted rotation axes**
- **Found during:** User testing (vertical/horizontal inverted)
- **Issue:** Both pitch and yaw rotating in wrong direction
- **Fix:** Negated both pitchChange and yawChange in `handleMouseMove()`
- **Committed in:** `d79000b`

**4. Rotation speed too fast**
- **Found during:** User testing (~30% faster than before)
- **Issue:** mouseSensitivity too high for quaternion-based rotation
- **Fix:** Reduced mouseSensitivity from 0.002 to 0.0014
- **Committed in:** `d79000b`
