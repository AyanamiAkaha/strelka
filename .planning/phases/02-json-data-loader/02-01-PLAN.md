<task type="auto">
  <name>Create JSON validator and parser</name>
  <files>src/core/validators.ts</files>
  <action>
    Create new file src/core/validators.ts with validation and parsing logic:

    1. Import JsonPoint from ./types and PointData from ./DataProvider

    2. Export validateJsonPoint(point: unknown, index: number): string | null
       - Check if point is non-null object, else return error
       - Cast to Record<string, unknown>
       - Check for required coordinates: x, y, z must exist
       - Type check: typeof x/y/z === 'number' (strict, no coercion)
       - Check cluster field if present: must be number or null
       - Return null if valid, error string otherwise

    3. Export parseJsonData(jsonText: string): PointData
       - Try: JSON.parse(jsonText) as unknown[]
       - Check if result is array, else throw "JSON must be an array of points"
       - Check array length > 30_000_000, else throw "Dataset too large"
       - Loop through array, call validateJsonPoint() for each point
       - If validation error, throw error with point index
       - On success, create Float32Array positions (length * 3) and clusterIds (length)
       - Fill buffers: positions[i*3]=x, positions[i*3+1]=y, positions[i*3+2]=z
       - Fill clusterIds: use cluster value or -1 (noise) if missing/null
       - Return PointData object with {positions, clusterIds, count: length}
       - Catch errors and console.error() before re-throwing

    Follow RESEARCH.md Pitfall 4 example for strict validation (no typeof checks for string "123")
    Follow RESEARCH.md Pitfall 1 for SyntaxError-specific catching
  </action>

  </verify>
  <done>Validator enforces strict types, parser converts to Float32Array</done>
