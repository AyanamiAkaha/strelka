<task type="auto">
  <name>Task 1: Create SQLite file loading utility</name>
  <files>src/core/DataProvider.ts</files>
  <action>
    Add loadSqliteFile() and getTableList() static methods to DataProvider.ts:

    1. Add import for validateTableSchema from validators.ts
    2. Create loadSqliteFile(file: File, tableName?: string): Promise<{ pointData: PointData, tables: string[] }> method:
       - Return Promise with FileReader.readAsArrayBuffer() reading
       - On reader.onload: create Uint8Array from result, create SQL.Database(uint8Array)
       - List tables with db.exec("SELECT name FROM sqlite_master WHERE type='table'")
       - If tables array is empty: throw Error('Database corrupt or unreadable')
       - If tableName not provided: return { pointData: empty, tables }
       - Validate schema with validateTableSchema(db, tableName)
       - Check row count with db.exec(`SELECT COUNT(*) FROM ${tableName}`)
       - If count > 30_000_000: throw Error(`Dataset too large: ${count} points (max 30,000,000)`)
       - Get hasCluster from validateTableSchema result
       - Prepare statement: db.prepare(`SELECT x, y, z${hasCluster ? ', cluster' : ''} FROM ${tableName}`)
       - Pre-allocate Float32Arrays: positions (count * 3), clusterIds (count)
       - Use db.each(stmt, {}, (row) => { /* fill arrays */ })
       - In each callback: positions[index*3] = row[0], positions[index*3+1] = row[1], positions[index*3+2] = row[2], clusterIds[index] = hasCluster ? (row[3] ?? -1) : -1
       - In done callback: stmt.free(), return { pointData: { positions, clusterIds, count }, tables }
       - Handle errors: reader.onerror (throw Error('Failed to read database file')), database errors (throw with descriptive message)

    3. Create getTableList(db: Database): string[] helper method:
       - Query sqlite_master: db.exec("SELECT name FROM sqlite_master WHERE type='table'")
       - Map values to string array of table names
       - Return array

    Pattern from RESEARCH.md: FileReader.readAsArrayBuffer() for binary files, db.prepare() + db.each() for incremental processing, stmt.free() in done callback (Pitfall 2, 3).

    Why: Each table extraction processes rows incrementally to avoid loading all data into memory at once. stmt.free() prevents memory leaks.
  </action>
  <verify>loadSqliteFile() reads SQLite file, lists tables, validates schema, extracts data to Float32Array, enforces 30M limit, throws "Database corrupt or unreadable" for corrupt files, throws "Dataset too large: {count} points (max 30,000,000)" for large datasets</verify>
  <done>loadSqliteFile() returns { pointData: PointData, tables: string[] } with WebGL-compatible buffers</done>
</task>
