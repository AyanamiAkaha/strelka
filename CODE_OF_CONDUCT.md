# Code of Conduct

This is a personal project maintained by a single author. The following principles apply:

**Technical merit is all that matters.** Contributions are evaluated solely on code quality, correctness, performance impact, and alignment with project goals. Nothing else.

**The author's decision is final.** This is a benevolent dictatorship. Architectural decisions, feature priorities, and acceptance criteria are set by the project maintainer. Disagreement is welcome in discussion; the final call is not up for debate.

**Be direct.** Say what you mean. Spare the pleasantries if you have a technical point to make. Concise, clear communication is valued over diplomatic padding.

**If you don't like it, fork it.** The BSD 3-Clause license explicitly permits this. You are free to take the code in any direction you choose under your own project.
